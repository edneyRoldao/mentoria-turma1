-> git init |   (init um repositorio, significa que as mudancas do diretoria serão controladas)

-> git status |     (verifica os arquivos alterados para comitar)

-> git add nome-do-arquivo |    (coloca o arquivo pra dentro da staging area)

-> git commit -m "mensagem para o commit" | (salva as mudancas, comita o codigo)

-> git log | (lista todos os commits)

-> git log --oneline | (lista todos os commits em uma linha)

-> git branch nome-da-branch |  (para criar uma branch, ramificacao)

-> git branch | (lista os branch do repositório)

-> git merge nome-da-branch | (para Juntar o codigo de duas branches)

-> git diff nome-do-arquivo | (exibe as mudancas que ocorreram no arquivo)

-> git add . (o ponto coloca todos na staging area)

-> git checkout nome-da-branch (troca de branch)

-> git checkout -b nome-da-nova-branch (cria a branch e faz o checkout)

-> git push (mando os commits locais para o remoto)

-> git pull (puxa os commits do remoto para o local)

-> git stash (salva os arquivos alterados em uma area de rascunho para serem usados depois)
